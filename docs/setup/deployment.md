# Deployment Guide

This document outlines the standard procedure for deploying the Richwell Portal to a production environment.

## Infrastructure Architecture

- **Web Server / Reverse Proxy:** Nginx
- **Backend Application Server:** Gunicorn (serving Django WSGI)
- **Frontend Serving:** Static files generated by Vite, served directly via Nginx
- **Database:** PostgreSQL with PostGIS extension
- **Process Management:** Supervisor or Systemd

## Deployment Steps

> [!CAUTION]
> Always ensure you have a database backup before running database migrations on a production environment.

### 1. Database Setup
Ensure PostgreSQL and PostGIS are installed and running. Create a new database and a dedicated user. Ensure the `postgis` extension is enabled on the database:
```sql
CREATE DATABASE richwell;
CREATE USER richwell_user WITH PASSWORD 'secure_pass';
ALTER ROLE richwell_user SET client_encoding TO 'utf8';
GRANT ALL PRIVILEGES ON DATABASE richwell TO richwell_user;
\c richwell
CREATE EXTENSION postgis;
```

### 2. Backend Deployment
Navigate to the `/backend` directory on your production server.

1. Create a Python virtual environment and activate it: `python -m venv venv && source venv/bin/activate`
2. Install dependencies: `pip install -r requirements.txt`
3. Configure your production `.env` file (see `environment.md`). Ensure `DEBUG=False`.
4. Run migrations: `python manage.py migrate`
5. Collect static files: `python manage.py collectstatic --noinput`
6. Compile translations (if applicable): `python manage.py compilemessages`
7. Start Gunicorn via Systemd or Supervisor.

### 3. Frontend Deployment
Navigate to the `/frontend` directory.

1. Install dependencies: `npm install`
2. Configure your production `.env` file to point `VITE_API_URL` to your production domain (e.g., `https://api.richwell.edu.ph/api`).
3. Build the static bundle: `npm run build`
4. The output will be located in the `/dist` folder. Copy this folder to your web root (e.g., `/var/www/richwell/frontend`).

### 4. Nginx Configuration
Configure Nginx to serve the frontend as static files and reverse-proxy `/api` / `/admin` requests to the Django Gunicorn socket.

> [!IMPORTANT]
> Ensure all traffic is forced over HTTPS using Let's Encrypt or another SSL provider.

```nginx
server {
    listen 80;
    server_name portal.richwell.edu.ph;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name portal.richwell.edu.ph;

    # SSL Config here...

    location / {
        root /var/www/richwell/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /admin/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /static/ {
        alias /var/www/richwell/backend/staticfiles/;
    }
}
```
