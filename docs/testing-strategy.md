# Testing Strategy

The Richwell Portal emphasizes reliability through a combination of fast Python Unit tests for the Backend logic and robust Playwright End-to-End (E2E) tests for the Frontend user journeys.

> [!IMPORTANT]
> The database should never be mutated by a test running on a production dataset. Always ensure `DEBUG=True` and test databases are dynamically generated during the test run.

## Backend (Python / Django)

We use `pytest-django`.

### How to Run
Navigate to `backend/` and run:
```bash
pytest
```

### Coverage
- **Models**: Validate custom `save()` methods, generation of unique IDNs (like `SystemSequence`), and constraints.
- **Views/Endpoints**: Test the DRF endpoints using Django's `APIClient`.
- **Permissions**: Assert that `403 Forbidden` is returned when a `STUDENT` accesses viewing a `PROFESSOR` profile, etc.

## Frontend (React)

We primarily use Playwright for End-to-End testing the complex form logic.

### How to Run Playwright E2E
Navigate to `frontend/` and run:
```bash
npm run test:e2e
npm run test:e2e:ui # For interactive debugging UI
```

### Coverage
- **Flow Tests (`academic_flow.spec.js` / `enrollment_flow.spec.js`)**: These files orchestrate full user journeys: from filling out an admission form, logging in as an admin to approve the form, then logging in as a student to pick subjects.
- **Assertion Strategy**: Tests click exactly what the user clicks. Avoid using DOM IDs (`#submit-btn`); instead, use roles `page.getByRole('button', { name: 'Submit' })`.

## Generating Seed Data

To test locally without writing automated files, we use the custom Django management commands:
- `python manage.py seed_applicants`
- `python manage.py seed_advising`
- `python manage.py seed_grade_submission`

(See `notes.txt` for exact data structures and login credentials generated by these seeds.)
